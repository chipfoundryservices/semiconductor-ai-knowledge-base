## What Is This?

A curated knowledge base of detailed technical articles, each 2,000-15,000 characters, covering the full stack from silicon to AI applications. Written for engineers, by engineers.

This is a subset of the 13,225+ entry knowledge base powering [ChipFoundryServices](https://www.chipfoundryservices.com/).

## Topics Covered

### Semiconductor Manufacturing
- EUV & High-NA lithography
- Etching (RIE, ALE, DRIE)
- Thin film deposition (CVD, PVD, ALD, epitaxy)
- CMP and planarization
- Ion implantation and doping
- Metrology and inspection
- Yield management and defect analysis
- Cleanroom technology

### Process Technology
- FinFET architecture and scaling
- GAA nanosheet transistors
- 3nm, 2nm, 1nm node challenges
- CFET (Complementary FET)
- Backside power delivery
- Interconnect scaling
- DTCO (Design-Technology Co-Optimization)

### Chip Design
- ASIC design flow
- RTL design and verification
- Synthesis and place-and-route
- Timing analysis and optimization
- DFT and test pattern generation
- Physical design and signoff
- ML-driven EDA tools

### AI & Machine Learning
- Large Language Models (LLMs)
- Transformer architecture
- Pre-training and fine-tuning (SFT, LoRA, QLoRA)
- RLHF and alignment
- RAG (Retrieval Augmented Generation)
- Inference optimization
- AI agents and tool use
- MCP (Model Context Protocol)

### GPU & Accelerators
- NVIDIA GPU architecture (Hopper, Blackwell)
- AMD MI300X
- Google TPU
- CUDA programming
- Tensor cores and mixed precision
- HBM (High Bandwidth Memory)
- Multi-GPU and distributed training

### Advanced Packaging
- 2.5D interposer (CoWoS)
- 3D stacking and hybrid bonding
- Chiplet architecture (UCIe)
- Fan-out packaging (InFO, FOWLP)
- Through-silicon vias (TSV)

## How to Use

Browse the `topics/` directory for individual articles, or search the full knowledge base at [chipfoundryservices.com](https://www.chipfoundryservices.com/).

For conversational exploration, try [CFSGPT](https://www.chipfoundryservices.com/chat/) — our AI assistant that can answer follow-up questions.

## File Format

Each topic file is a markdown document with:
- Bold title and definition
- 4-6 sections with bullet points
- Specific numbers, vendor names, and benchmarks
- No fluff — production-grade technical content
